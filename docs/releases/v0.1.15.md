# v0.1.15

## 本版亮点

- 修复部分出售帖里“购买框位置会在购买前后跳动”的问题，未购买和购买后现在都会保持和原站一致的内容顺序。
- 像 `tid-2894068` 这类帖子，购买框、隐藏内容和图片不再被错误重排，阅读时不会再出现“刚买完位置突然跑掉”的割裂感。

## 体验优化

- 帖子正文现在会尽量忠实保留论坛原始 DOM 的段落顺序，出售框、引用、图片和普通文本会一起按原位渲染。
- 出售框组件被抽成统一视图，正文流里的出售块和帖子外层购买交互共用同一套样式与按钮逻辑，后续维护和扩展更稳。
- 补上了针对“出售框在图片前”“不重复渲染”“购买按钮仍可正常触发”的回归测试，后续再改帖子渲染链时更不容易把这个问题带回来。

## 修复

- 修复未购买状态下会把 `h6.quote.jumbotron` 出售框从正文中抽离、再按“最前/最后”二选一重新拼装，导致显示顺序和原帖不一致的问题。
- 修复购买成功刷新后，帖子改按正文原始顺序渲染，因而和购买前产生明显跳位的问题。
- 修复出售框进入正文渲染链后，相关 `blockquote` 提示内容可能被当成普通引用重复渲染的问题。
- 修复附件搬运与正文顺序渲染并存时，个别帖子会多出多余空行的文本归一化问题。

## 技术细节

- `ThreadContentParser` 现在会把出售框识别为正式的内容段，而不是只作为正文外的附加元数据。
- `ThreadDetailParser` 改为分别生成“顺序渲染副本”和“纯文本/购买信息提取副本”，既保留出售框原始位置，也不破坏现有购买流程。
- `ThreadPostRenderModel` 和 `ThreadRichContent` 新增出售框渲染块，帖子正文可以像处理文本、图片、引用一样处理购买区块。
- `ThreadPostBody` 会优先信任预编译后的正文块顺序；如果正文里已经含有出售框，就不再额外补一遍前置/后置购买框。

## 发布说明

- 本版本继续由 GitHub Actions 自动构建 Android、iOS 和 Windows 安装包。
- Release 页面会继续提供 Android 多架构 APK、AAB、iOS sideload IPA、Windows x64 压缩包和 SHA256 校验文件。
