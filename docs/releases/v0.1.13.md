# v0.1.13

## 本版亮点

- 任务中心在“旧完成记录已经过期，但新一轮奖励已经可领”时，会继续检查进行中任务奖励，减少周常、日常奖励漏领。
- README 和发布页的构建状态徽章已改为准确反映当前 release 结果，不再长期误报 `failing`。

## 体验优化

- 发布页里的构建徽章现在会直接跳到对应的 GitHub Actions 运行记录，排查发布状态更直接。
- 仓库首页的发布状态改为独立镜像流程，不再被旧 tag 的历史运行结果污染。

## 修复

- 修复旧周期任务完成时间会阻断当前周期奖励领取检查的问题。
- 修复 release badge 读取默认 workflow 历史视图后长期显示失败的问题。

## 发布说明

- 本版本继续由 GitHub Actions 自动构建 Android、iOS 和 Windows 安装包。
- Release 页面会继续提供 Android 多架构 APK、AAB、iOS sideload IPA、Windows x64 压缩包和 SHA256 校验文件。
